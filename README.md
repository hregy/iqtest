# IQ Test (Mobile Web PWA)

A mobile-first, image-based IQ test. Each attempt draws **20 random questions**
from a pool of **100**, gives **10 seconds per question** (auto-advances), is
**unreversible** (no going back), and applies **anti-copy / anti-screenshot
deterrents**.

## Stack
- React + Vite + TypeScript
- Hand-rolled PWA (offline via `public/sw.js` + `public/manifest.webmanifest`)
- Questions are self-contained SVG images generated by a Python script

## Develop / build
```bash
npm install
npm run dev        # local dev server (http://localhost:5173)
npm run build      # production build -> dist/
npm run preview    # serve the production build
npm run generate   # regenerate the 100 question images + manifest
```

## How questions work
Each question is one SVG in [public/questions/](public/questions/) with the four
answer choices (A–D) drawn into the image. The app only renders A/B/C/D tap
buttons; the correct choice is stored in [src/data/questions.json](src/data/questions.json):

```json
{ "id": "q001", "image": "/questions/q001.svg", "type": "matrix-reasoning",
  "category": "pattern", "options": ["A","B","C","D"], "correctIndex": 0 }
```

Question types (all purely visual):
- `matrix-reasoning` — 3×3 Raven's-style "find the missing piece" → `pattern`
- `visual-analogy` — "A is to B as C is to ?" → `analogy`
- `odd-one-out` — "which one is the odd one out?" → `spatial`
- `shape-progression` — "which shape completes the pattern?" → `series`

Categories drive the per-category breakdown on the results screen and are
data-driven, so adding a new category just works.

Every question is validated by `npm run verify`, which independently re-derives
each answer and flags any ambiguous or inconsistent puzzle.

### Swapping in your own questions
Two options:
1. **Replace the generator** — edit
   [scripts/generate_questions.py](scripts/generate_questions.py) and run
   `npm run generate`.
2. **Use your own images** — drop image files into `public/questions/` and edit
   `src/data/questions.json` so each entry points at your file with the correct
   `correctIndex` (0–3). Keep the pool at 100 (or change `POOL_SIZE`).

## Configuration
All knobs live in [src/config.ts](src/config.ts):
- `TEST_LENGTH` (20) — questions per attempt
- `QUESTION_SECONDS` (10) — time per question
- `ONE_ATTEMPT_LOCK` (false) — set `true` to let a device take the test only once
  (soft lock via `localStorage`; a user can clear it)

## Protections — what's real vs. best-effort
| Requirement | Status |
|---|---|
| 20 random of 100 | ✅ Fisher–Yates pick at start |
| 10s per question, auto-advance | ✅ Per-question countdown |
| Unreversible (no back) | ✅ Forward-only state + browser Back trapped (`popstate`) |
| No copy / paste / text select | ✅ `user-select:none`, context menu / copy / drag blocked |
| No screenshot | ⚠️ **Deterrent only** |

> **Screenshots cannot be truly blocked on the web.** Browsers expose no API to
> stop OS-level screen capture. This app does the best a web app can: it blurs
> the question behind a warning overlay whenever the tab is hidden or the window
> loses focus (when most capture/lookup happens), disables long-press image
> saving, and intercepts common capture/devtools shortcuts. To *truly* block
> screenshots you need a native app (Android `FLAG_SECURE`).

## Disclaimer
The score is a gamified estimate for a fast visual quiz — **not** a validated
psychometric IQ measurement.
